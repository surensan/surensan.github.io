# 作品集维护说明

## 最简单的新增方法

不用改 `app.js`，也不用动网页结构。

1. 打开这个文件夹：

```text
F:\Codex\作品集\assets\new-details
```

2. 新建一个作品文件夹。最省事的命名方式是：

```text
详情页-儿童学习桌
主图-儿童学习桌
视觉规范-甘源双十一
产品渲染-吹风机
合成海报-坚果礼盒
PPT-品牌手册
```

脚本会自动读取前缀，把作品放进对应分类；后面的文字就是作品标题。

3. 把图片放进去，按展示顺序命名：

```text
01.jpg
02.jpg
03.jpg
```

4. 如果只靠文件夹名不够，再在同一个作品文件夹里新建 `meta.txt`，照这个格式填：

```text
标题=儿童学习桌详情页
主分类=平面设计
分类=详情页
版权方=品牌名
显示分类=电商详情页
标签=详情页,电商页面,儿童产品
修改记录=2026-06
显示记录=12
```

5. 回到根目录，双击：

```text
publish.bat
```

它会自动更新作品清单，再上传到 GitHub Pages。等 1-3 分钟后刷新网站即可。

## 可用分类

主分类可以写：

```text
平面设计
渲染作品
```

分类可以写：

```text
详情页
主图
视觉规范
AI 海报
合成海报
产品渲染
详情页卖点渲染
三维静态
三维动态
练习作品
动画展示
PPT
```

也可以直接把分类写在文件夹名前面，比如 `详情页-作品名`。常用前缀：`详情页`、`主图`、`视觉规范`、`首页规范`、`电商首页`、`合成海报`、`产品渲染`、`三维静态`、`详情页卖点渲染`、`三维动态`、`练习作品`、`动画展示`、`PPT`。

脚本会把这些中文分类转换成网页里已有的分类代号，所以不会破坏原来的筛选结构。

## 可选字段

如果不想用第一张图当封面：

```text
封面=02.jpg
```

默认不写简介。需要说明文字时，直接做进作品图片里再更新。

## 首页精选顺序

首页精选作品由 `app.js` 里的 `homeProjectMap` 控制。以后新增到首页精选时，把新的作品 ID 放在对应数组的第一个位置。

如果详情弹窗想用网格展示：

```text
详情布局=grid
```

## 只更新清单，不上传

本地预览前可以只运行：

```powershell
cd F:\Codex\作品集
.\update-projects.ps1
```

## 不想显示某次小修改

不要把那个月份写进 `修改记录` 即可。例如只想显示 6 月和 8 月：

```text
修改记录=2026-06,2026-08
```

## 多张图合并成一个项目

如果卖点渲染、KV 渲染里几张图其实属于同一个项目，不要分开放多个文件夹。直接放进同一个作品文件夹：

```text
产品渲染-吹风机卖点渲染
  01.jpg
  02.jpg
  03.jpg
  meta.txt
```

这样作品库里只显示一个卡片，点开后按 01、02、03 的顺序展示。

如果是已经写在 `app.js` 里的旧项目，需要把要合并的作品标题发给我，我会把它们整理到同一个项目的 `detailImages` 里。

## 手动替换旧封面

如果只是替换原有项目封面，保持同一个文件路径即可。例如：

```text
assets/covers/detail-page-17.jpg
```

平时改布局、改文案、改自动上传脚本，不会覆盖这些封面图。
