# 作品集维护说明

## 最简单的新增方法

不用改 `app.js`，也不用动网页结构。

1. 打开这个文件夹：

```text
F:\Codex\作品集\assets\new-details
```

2. 新建一个作品文件夹，文件夹名可以直接用作品名。

3. 把图片放进去，按展示顺序命名：

```text
01.jpg
02.jpg
03.jpg
```

4. 在同一个作品文件夹里新建 `meta.txt`，照这个格式填：

```text
标题=儿童学习桌详情页
主分类=平面设计
分类=详情页
标签=详情页,电商页面,儿童产品
简介=儿童学习桌产品详情页，围绕成长场景、收纳结构和安全材质展开。
修改记录=2026-06
显示记录=12
```

5. 回到根目录，双击：

```text
publish.bat
```

它会自动更新作品清单，再上传到 GitHub Pages。等 1-3 分钟后刷新网站即可。

## 可用分类

主分类可以写：

```text
平面设计
渲染作品
```

分类可以写：

```text
详情页
主图
视觉规范
AI 海报
合成海报
产品渲染
详情页卖点渲染
练习作品
动画展示
```

脚本会把这些中文分类转换成网页里已有的分类代号，所以不会破坏原来的筛选结构。

## 可选字段

如果不想用第一张图当封面：

```text
封面=02.jpg
```

如果想自定义卡片上那句短说明：

```text
卡片说明=把产品卖点组织成一条清晰的购买路径。
```

如果详情弹窗想用网格展示：

```text
详情布局=grid
```

## 只更新清单，不上传

本地预览前可以只运行：

```powershell
cd F:\Codex\作品集
.\update-projects.ps1
```

## 不想显示某次小修改

不要把那个月份写进 `修改记录` 即可。例如只想显示 6 月和 8 月：

```text
修改记录=2026-06,2026-08
```

## 手动替换旧封面

如果只是替换原有项目封面，保持同一个文件路径即可。例如：

```text
assets/covers/detail-page-17.jpg
```

平时改布局、改文案、改自动上传脚本，不会覆盖这些封面图。
